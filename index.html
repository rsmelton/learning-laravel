My experience with Laravel, SQL, and ADF

To learn Laravel, I decided to build a pre-existing application that I had previously built using the MERN stack, but this time with Laravel.

The app can be found here: https://dragons-brew-cafe-45mgn.sevalla.app

App Features:

  • Registration and login with Laravel Breeze
  • User can add and remove items from their cart
  • Cart counter updates immediately to reflect users actions through the use of Alpine.js
  • Uses SQL database to store users and their respective cart items

Through building this app I learned:

  • How to build Blade components
  • How to write and use migrations to interact with SQL database
  • How to build and use database seeders
  • How to build controllers to handle requests from the frontend i.e handling validation, fetching from the DB with SQL queries, CRUD operations, etc.
  • About the MVC architecture of Laravel:
    • Model: File that represents an entity that will be stored in the database. Laravel uses Eloquent models meaning we can define relationships in these files that represent how entities interact with each other. Ex. A user can have many cart items while a cart item belongs to a user.
    • View: Simply what gets displayed to the user through Blade comonents. Blade components are essentially just reusable components to be used in the frontend of the application.
    • Controller: Handles the heavy lifting of the application whether thats calculations, database queries, or handling requests from the frontend and providing a response.
  • How to write SQL queries:

    While these are basic examples, I didn't need very complex queries in this application.

    Outside the code:
    • Ex. Get all users: SELECT * FROM users;
    • Ex. Get specific users cart items: SELECT * FROM cart_items WHERE id = 1;
    Inside the code:
    • Ex. Grabbing all the authenticated users cart items: $userCartItems = CartItem::where('user_id', auth()->id())->get();
    • Ex. Checks to see if a specific menu item already exists in the users cart to know whether to create the new item, or to simply increment its quantity: $existingCartItem = CartItem::where('user_id', $userId) ->where('menu_item_id', $menuItemId) ->first();

ADF:

While it didn't make any sense to use ADF in this application, I figured it would still be a good idea to try and get a better understanding of how it works.

The way I understood it well was by understanding this analogy: Imagine a chocolate factory and we have a manager and this manager is responsible for gathering the needed resources to make the chocolate. Once the manager has all the resources he can then pass those resources off to the workers and these workers can clean, mix, and wrap the chocolate. Once we have the chocolate we can then send it off to our store. Using this analogy we can understand that our resources could be different types of files that hold data. The manager is ADF and ADF passes off these files to its workers to clean the data or complete other activities. Once these activities are completed we can then transfer the data to be loaded into our database. Another easy way to remember this is that ADF uses ETL or Extract, Transfer, Load.
